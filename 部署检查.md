# 部署前检查清单

## 必须检查的项目

### 1. 文件完整性
- [ ] `index.html` 存在
- [ ] `main.js` 存在
- [ ] `style.css` 存在
- [ ] `image/` 文件夹存在且包含所有图片

### 2. 图片路径
检查 `main.js` 中的图片路径：
- 应该使用相对路径：`image/微信图片_xxx.jpg`
- 不要使用绝对路径
- 确保路径大小写正确

### 3. CDN 可用性
- 代码已包含多个 CDN 备用
- 如果所有 CDN 都失败，考虑使用本地版本

### 4. GitHub 提交
```bash
# 检查文件状态
git status

# 确保所有文件已添加
git add .

# 提交
git commit -m "部署准备"

# 推送
git push
```

### 5. GitHub Pages 设置
- Settings > Pages
- Source: main branch, / (root)
- 等待部署完成

## 常见错误及解决

### 错误：404 Not Found
**原因**：文件路径错误或文件未上传
**解决**：检查所有文件是否在仓库中

### 错误：Failed to load resource
**原因**：CDN 无法访问或图片路径错误
**解决**：
1. 检查图片路径
2. 尝试使用本地 Three.js

### 错误：CORS 错误
**原因**：跨域问题
**解决**：GitHub Pages 应该自动处理，如果还有问题，检查文件路径

## 快速修复

如果部署失败，尝试：

1. **检查图片是否上传**：
   ```bash
   git ls-files image/
   ```

2. **使用本地 Three.js**：
   ```bash
   ./download-three.sh
   # 然后修改 index.html 使用 index-local.html
   ```

3. **检查文件大小**：
   ```bash
   du -sh image/*
   ```

4. **重新部署**：
   - 在 GitHub 仓库页面
   - Settings > Pages
   - 点击 "Redeploy"
