# 调试指南

## 如果页面无法打开或显示错误

### 步骤 1：检查服务器是否运行

在终端运行：
```bash
lsof -ti:8000
```

如果有输出，说明服务器正在运行。如果没有输出，运行：
```bash
python3 -m http.server 8000
```

### 步骤 2：检查浏览器控制台

1. 按 `F12` 或 `Cmd+Option+I`（Mac）打开开发者工具
2. 点击 **Console** 标签页
3. 查看红色错误信息
4. 复制错误信息

### 步骤 3：常见错误及解决方案

#### 错误 1：`Failed to load resource` 或 `ERR_CONNECTION_RESET`
**原因**：无法加载 Three.js CDN
**解决**：
- 检查网络连接
- 尝试刷新页面（会自动尝试备用 CDN）
- 或运行 `./download-three.sh` 下载本地版本，然后使用 `index-local.html`

#### 错误 2：`Cannot read property 'xxx' of null`
**原因**：某个元素未找到
**解决**：确保 HTML 文件完整，所有 ID 都存在

#### 错误 3：`Shader Error` 或 `WebGL Error`
**原因**：着色器编译错误或 WebGL 不支持
**解决**：
- 使用现代浏览器（Chrome、Firefox、Edge、Safari）
- 检查浏览器是否支持 WebGL：访问 https://get.webgl.org/

#### 错误 4：页面空白
**原因**：可能是 JavaScript 错误导致初始化失败
**解决**：
- 查看控制台错误信息
- 确保使用服务器运行（不能直接打开文件）
- 检查 `main.js` 文件是否完整

### 步骤 4：测试步骤

1. **测试服务器**：
   访问 http://localhost:8000/test.html
   如果能看到"测试页面"，说明服务器正常

2. **测试主页面**：
   访问 http://localhost:8000/index.html
   查看控制台错误

3. **检查文件**：
   确保以下文件存在：
   - index.html
   - main.js
   - style.css

### 步骤 5：获取帮助

如果问题仍然存在，请提供：
1. 浏览器控制台的完整错误信息
2. 使用的浏览器和版本
3. 操作系统信息
